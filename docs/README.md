# Frontend Documentation

This folder is the documentation hub for the frontend repo.

## Start Here

- `EVIDENCE_APP_ARCHITECTURE.md` - Evidence app route, page, context, auth, and API-client map.
- `CUSTOMER_HELP_FUNCTIONALITY_GUIDE.md` - customer-facing help content for the Help section and Axiom.
- `CI_CD_PIPELINE.md` - frontend CI/CD behavior and deployment optimization notes.
- `DEPLOYMENT.md` - detailed EC2 deployment guide.
- `DEPLOYMENT_PROTOCOL.md` - short source-of-truth deployment protocol.
- `REPOSITORY_STRATEGY.md` - cross-repo coordination and future repository split notes.

## Documentation Rule

Keep customer-facing help free of backend implementation detail. If the user only needs to know what to click or what a status means, document that in `CUSTOMER_HELP_FUNCTIONALITY_GUIDE.md`. Put build, CI, Docker, and incident details in deployment docs.
